# Day 15: Inception、RoBERTa、VGG 与 GPT-1

## 第一部分：GoogLeNet / Inception

### 一、传统方法的局限

增加网络深度和宽度 → 参数过多 → 过拟合 + 计算量暴增。

### 二、根本解决思路

从全连接架构 → 稀疏连接（分组卷积），极大减少计算量。

问题：稀疏连接导致频繁间接选址 + 缓存不命中。

### 三、Inception 模块

利用多种方式（不同卷积核 + 池化）处理前一层后拼接：

![Inception](https://i.postimg.cc/bJmn3nWN/image.png)

在计算需求过高的地方用 1×1 卷积缩减通道数，同时让信息在不同尺度上得到处理。

### 四、实践要点

| 要点 | 说明 |
|------|------|
| 低层用传统卷积 | 高层才用 Inception 模块，更为有效 |
| 核心优势 | 每一层显著增加单元数量而计算复杂性不爆炸 |
| R-CNN 配合 | 先锁定物体位置 → Inception 精细分类 |

## 第二部分：RoBERTa

> 核心结论：BERT 训练不足。

### 一、四大改进

| 改进 | BERT | RoBERTa |
|------|------|------|
| 数据 | — | 更大批、更多数据、更长时间 |
| NSP | 有 | 移除 |
| 序列 | 128→512 两阶段 | 一直用 T=512 |
| 掩码 | 静态（预处理一次） | 动态（每次输入重新掩码） |

### 二、静态掩码 vs 动态掩码

| | 静态（BERT） | 动态（RoBERTa） |
|------|------|------|
| 做法 | 预处理时掩码一次，复制 10 份不同掩码方式，训 40 轮 | 每次输入模型时重新生成掩码 |
| 问题 | 模型可能记住掩码位置 | 无 |

### 三、NSP 消融实验

| 格式 | 说明 | 结果 |
|------|------|------|
| SEGMENT-PAIR（BERT） | 句子对 | 下游有损 |
| SENTENCE-PAIR | 单个句子 | 更差（无法学长距离依赖） |
| FULL-SENTENCES | 连续完整句子（可跨文档） | 最优 |
| DOC-SENTENCES | 不跨文档 | 批量大小不固定 |

> 去掉 NSP + 保留 SEGMENT-PAIR 输入格式 → 提升下游性能。

### 四、其他改进

| 改进 | 说明 |
|------|------|
| 大批量训练 | 优化困惑度，提高准确性 |
| 字节级 BPE | 无需预处理，虽性能略降但优势明显 |
| 更多数据 + 更长时间 | 带来明显提升 |

## 第三部分：VGG

### 一、核心设计：小卷积核堆叠

| | 大卷积核（7×7） | 小卷积核堆叠（3×3×3） |
|------|------|------|
| 感受野 | 相同 | 相同 |
| 参数量 | 多 | 少 → 正则化效果 |
| 非线性 | 1 层 | 3 层 → 拟合更复杂特征 |

> 引入 1×1 卷积：增加非线性同时不影响感受野。

### 二、训练策略

| 策略 | 做法 |
|------|------|
| 预处理 | 每个像素减去训练集平均 RGB 值 |
| 优化器 | 小批量梯度下降 + 动量 |
| 正则化 | 权重衰减 + 前 2 层全连接 Dropout |
| 学习率 | 初始 10⁻²，准确率不再提升时 ÷10 |
| 初始化 | 先训小网络 → 用其权重初始化大网络 |

### 三、训练尺度

| 方式 | 做法 | 优势 |
|------|------|------|
| 单尺度 | 最短边统一缩放 → 裁剪 | 不同裁剪位置 = 多尺度特征 |
| 多尺度 | 每次迭代随机选尺度范围 | 模型看到不同大小物体 |

### 四、测试：密集评估 vs 多裁剪

| | 多裁剪（AlexNet） | 密集评估（VGG） |
|------|------|------|
| 做法 | 剪出几十个 224×224 块分别输入 | 整张图一次输入全卷积网络 |
| 效率 | 极低（大量重复计算） | 极高 |
| 填充 | 0 填充（丢失上下文） | 邻居填充（更大感受野） |

> 两者互补，边界填充方式不同，关注特征侧重点不同，可结合使用。

将全连接层替换为 7×7 + 1×1 + 1×1 卷积层 → 不受输入尺寸限制。

## 第四部分：GPT-1

> Improving Language Understanding by Generative Pre-Training

### 一、动机

标注数据稀缺 → 无监督预训练 + 监督微调。

### 二、无监督预训练

$$L_1(\mathcal{U}) = \sum_i \log P(u_i \mid u_{i-k}, \ldots, u_{i-1}; \Theta)$$

- 最大化在前 k-1 个词条件下正确预测下一词的概率
- 自回归语言模型（单向）

#### 输入构建

$$h_0 = U W_e + W_p$$

| 符号 | 含义 |
|------|------|
| $U$ | 输入 token 索引（上下文窗口 k） |
| $W_e$ | Token Embedding 矩阵 |
| $W_p$ | 位置嵌入矩阵 |
| $+$ | 逐元素相加（词嵌入 + 位置嵌入） |

#### Transformer 前向

$$h_l = \text{transformer\_block}(h_{l-1}), \quad l = 1, \ldots, n$$
$$P(u) = \text{softmax}(h_n W_e^T)$$

### 三、监督微调

$$P(y \mid x^1, \ldots, x^m) = \text{softmax}(h_l^m W_y)$$
$$L_2(\mathcal{C}) = \sum_{(x,y)} \log P(y \mid x^1, \ldots, x^m)$$

#### 辅助目标

$$L_3(\mathcal{C}) = L_2(\mathcal{C}) + \lambda \cdot L_1(\mathcal{C})$$

> 把语言建模作为辅助目标：提升泛化能力、加快收敛、提高性能。

### 四、任务输入转换

将结构化输入转换为有序序列：

| 任务 | 做法 |
|------|------|
| 文本蕴含 | 前提 + 分隔符 + 假设 |
| 相似性 | 两句子双向拼接（消除顺序影响） |
| 问答 | 上下文 + 问题 + 分隔符 + 候选答案 |

### 五、关键发现

- 每一层（包括 Embedding）都能带来性能提升
- Transformer 的归纳偏置（全局交互 + 位置编码）有助于迁移
- 大数据集更能从辅助目标中受益
- 预训练影响所有任务的性能

---

## 五、要点总结

1. **Inception**：多卷积核并行 + 1×1 降通道，稀疏连接减少计算
2. **RoBERTa**：BERT 训练不足 → 动态掩码、去 NSP、更大批次
3. **VGG**：小卷积核堆叠替代大核，密集评估比多裁剪快且上下文更丰富
4. **GPT-1**：无监督预训练 LM + 监督微调 + 辅助 LM 目标，奠定 GPT 系列基础
