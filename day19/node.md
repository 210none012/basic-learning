# Day 19: LoRA —— 低秩适配与高效微调

## 一、背景与动机

### 问题

模型规模越来越大（GPT-3 175B，Llama 70B，DeepSeek 等），全量微调（Full Fine-Tuning）成本急剧增高：

- **显存需求**：全量微调需要存储模型参数 + 优化器状态 + 梯度，175B 模型需要 ～1.4TB 显存
- **部署成本**：每个下游任务需要保存一份完整模型，存储与切换开销巨大
- **时间成本**：全量反向传播计算量随参数量线性增长

### 已有方案及其局限

| 方法 | 核心思路 | 局限 |
|------|---------|------|
| **Adapter** | 在 Transformer 层中插入小型可训练瓶颈模块 | 必须顺行计算，无法与主干并行，增加推理延迟 |
| **Prefix/Prompt Tuning** | 为适应任务在输入序列前插入可训练的特殊 token | 占用有效序列长度，难以处理长上下文任务 |
| **BitFit** | 仅微调 bias 项 | 表达能力有限，复杂任务效果不佳 |

## 二、LoRA 核心思想

### 假设：模型适应任务的更新量是低秩的

过参数化（over-parameterization）的神经网络虽然在参数量上远大于数据点数量，但训练过程中**梯度下降的动力学路径**天然倾向于收敛到**低秩解**。这意味着微调时权重的变化量 $\Delta W$ 可以用低秩矩阵分解来近似。

### 数学形式

原始前向传播：$h = W_0 x$
LoRA 前向传播：$h = W_0 x + \underbrace{BA x}_{\Delta W x}$
其中：
-  $\in \mathbb{R}^{d \times k}$：预训练权重，**冻结不变**
-  $\in \mathbb{R}^{d \times r}$， $\in \mathbb{R}^{r \times k}$：可训练的低秩矩阵
-  $\ll \min(d, k)$：秩，通常取值 4、8、16、64

最终：$h = W_0 x + \frac{\alpha}{r} \cdot BA x$

![LoRA 结构示意](https://i.postimg.cc/4yj098p4/image.png)

### 初始化和缩放

| 参数       | 初始化方式   | 原理                               |
| -------- | ------- | -------------------------------- |
| A        | 随机高斯初始化 | 提供初始探索方向                         |
| B        | 全零初始化   | 确保 $\Delta W$ 初始为 0，训练从预训练模型状态开始 |
| $\alpha$ | 缩放常数    | = W_0 x + \frac{\alpha}{r} BA x$ |

**关于 $\alpha$ 的关键理解**：

- 改变 $\alpha$ 的效果 **≈ 改变学习率**
- 只要 A 的初始化幅度与 $\alpha$ 协调，$\alpha$ 本质上就是**学习率的缩放因子**
- 实际调参时：设定 $\alpha$ 后，调整 r 即可，减少重新调参的需求

### 参数量对比

以 GPT-3 175B 为例：

| 方法 | 可训练参数 | 压缩比 |
|------|----------|--------|
| 全量微调 | 175B | 1× |
| LoRA (r=4) | ～37M | ～4700× |
| LoRA (r=16) | ～150M | ～1100× |

## 三、LoRA 的关键优势

### 1. 无额外推理延迟
- 部署时可将 BA 合并到 W 中
- 合并后与原始模型结构完全一致，推理速度为原始速度
- 不合并时可在不同任务间快速切换 LoRA 模块

### 2. 共享预训练模型
- 一个预训练模型可共享给多个任务
- 每个任务只需存储独立的 LoRA 模块（几 MB 到几百 MB）
- 硬件门槛显著降低：175B 模型在单张 A100 上也可完成

### 3. 兼容性
- 与其他微调方法（Adapter、Prefix Tuning）不冲突，可组合使用
- 可与量化技术（QLoRA）结合进一步节省显存

### 4. 简单线性设计
- 仅需修改前向传播的计算公式
- 不引入任何非线性层或额外模块

## 四、LoRA 作用的权重矩阵


> **核心结论**：相比于以更大秩只适配单一类型的权重，更倾向于适配更多的权重矩阵。

## 五、实践要点总结

| 维度           | 建议                                    |
| ------------ | ------------------------------------- |
| **目标矩阵**     | $W_q$ 和 $W_k$ 即可满足多数场景                |
| **秩 r**      | 4～16 之间，一般 8 开始调，但在实验中秩为1也显示出良好效果     |
| **$\alpha$** | 通常 2×r、4×r，本质是学习率缩放                   |
| **合并策略**     | 频繁切换任务 → 分离模式；线上部署 → 合并模式             |
| **组合使用**     | LoRA + Prefix Tuning / Adapter 可进一步提升 |
