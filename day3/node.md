# Day 3: RAG（检索增强生成）学习笔记及环境部署

## 一、RAG 概述

### 什么是 RAG

RAG（Retrieval-Augmented Generation）通过**参考外部知识库**来增强 LLM 的回答能力，有效减少模型在处理未知知识时产生的幻觉。

### 发展轨迹

1. **早期**：通过预训练模型（PTM）将额外知识融入语言模型，重在完善预训练技术
2. **中期**：LLM 展现出强大上下文能力后，转向为 LLM 提供更好的信息
3. **现在**：RAG 增强不再局限于推理阶段，与微调相融合

## 二、朴素 RAG（Naive RAG）

### 核心流程：索引 → 检索 → 生成

| 阶段 | 说明 |
|------|------|
| **索引** | 原始数据清洗提取 → 转换为纯文本 → 分割成块 → Embedding 向量化 → 存入向量数据库 |
| **检索** | 用户查询向量化 → 计算与语料库向量相似度 → 取 Top K 最相关块 |
| **生成** | 查询 + 检索文档 → 组合成 Prompt → LLM 生成答案 |

### 三大问题

- **检索挑战**：检索到的块可能与问题不匹配
- **生成困难**：幻觉、内容无上下文支持、不相关性、毒性、偏见
- **增强障碍**：多来源导致信息冗余/重复；单次检索信息不足

---

## 三、高级 RAG（Advanced RAG）

### 方向：提升检索质量

#### 预检索（优化索引 + 查询）

| 策略 | 方法 |
|------|------|
| 优化索引 | 增强数据粒度、优化索引结构、添加元数据、对齐优化、混合检索 |
| 查询优化 | 查询重写、查询转换、查询扩展 |

#### 后检索（集成上下文）

| 策略 | 方法 |
|------|------|
| 重新排序 | 将最相关内容移到 Prompt 前端 |
| 上下文压缩 | 强调关键信息，缩短上下文长度 |

## 四、模块化 RAG（Modular RAG）

### 新增模块

| 模块 | 功能 |
|------|------|
| **Search** | 跨数据源搜索 |
| **Fusion** | 查询扩展为多视角，并行向量搜索 |
| **Memory** | 利用 LLM 内存指导检索 |
| **Routing** | 为查询选择最佳数据源路径 |
| **Predict** | LLM 直接生成上下文，减少冗余噪声 |
| **Task Adapter** | 适配各种下游任务 |

### 新模式

| 模式 | 说明 |
|------|------|
| Rewrite-Retrieve-Read | LLM 优化用户问题后再检索 |
| Generate-Read | 用 LLM 生成内容替代传统检索 |
| Recite-Read | 从模型权重检索知识 |
| 混合检索策略 | 关键词 + 语义 + 向量搜索 |
| HyDE | 假设文档嵌入 |

## 五、RAG vs 微调

| 维度 | RAG | 微调 |
|------|-----|------|
| 知识更新 | 实时，无需重训 | 静态，需重新训练 |
| 可解释性 | 高（可溯源文档） | 低 |
| 计算资源 | 较低 | 高（需大量训练） |
| 延迟 | 较高 | 较低 |
| 定制深度 | 浅（依赖外部知识） | 深（改变模型行为） |

> 两者不互斥，可以结合使用。

## 六、检索详解

### 检索源类型

| 类型 | 示例 | 处理方式 |
|------|------|----------|
| 非结构化 | 纯文本 | 直接向量化 |
| 半结构化 | PDF（文本+表格） | SQL 查询 或 表格转文本 |
| 结构化 | 知识图谱 | 精确查询 |
| LLM 生成 | GenRead, SKR | 模型生成替代检索 |

### 检索粒度

| 粒度 | 特点 |
|------|------|
| Token | 精细但计算量大 |
| 短语 | 比 Token 更语义完整 |
| 句子 | 平衡粒度 |
| 块（Chunk） | 最常用，含丰富上下文 |
| 文档 | 信息全但噪声多 |

### 检索方法混合

- 稀疏检索（BM25）+ 密集检索（Embedding）
- 混合检索 = 关键词 + 语义 + 向量

## 七、增强过程

### 三种增强检索方式

![检索方式](https://i.postimg.cc/QxkSbwV4/ping-mu-jie-tu-2026-07-01-132618.png)

| 方式 | 原理 | 适用场景 | 代表方法 |
|------|------|----------|----------|
| **迭代检索** | 基于已生成文本反复搜索 | 需要上下文补充 | Iter-RetGen |
| **递归检索** | 根据结果细化查询，反馈循环 | 需求不明确/多跳推理 | IRCoT, ToC |
| **自适应检索** | LLM 主动判断何时检索 | 灵活场景 | Flare, Self-RAG, WebGPT |

## 八、REALM（检索增强语言模型预训练）

### 核心思想

使用无监督文本的性能信号来训练检索器：**改善困惑度的检索应被奖励，信息量不足的检索应被惩罚**。

### 数学形式

$$p(y \mid x) = \sum_{z \in \mathcal{Z}} p(y \mid z, x) \, p(z \mid x)$$

- Z：语料库
- z：检索的文档
- x：输入
- y：输出（被 mask 的词）

### 流程

1. 内积计算文档相关性 → Softmax 归一化 → 得到 p(z|x)
2. BERT 提取 x 和 z 的语义并压缩
3. 拼接 x 和 z 输入 Transformer 预测被 mask 位置
4. 所有位置概率相乘 = 完全正确概率（预训练目标）
5. 计算 z 中与 y 匹配片段的起止语义向量
6. 拼接 → 全连接层 → 得分

### 工程优化

- p(y|x) 计算量过大 → **取 Top K 文档近似求和**
- 内积每次反向传播都变化 → **异步刷新索引**
- 训练初期 Embedding 差 → **预热（Warm Start）**：给定句子，训练模型检索其来源文档

## 九、评估指标

### 检索质量

| 指标 | 说明 |
|------|------|
| 命中率 (Hit Rate) | 相关文档是否在 Top K 中 |
| MRR | 第一个相关文档排名的倒数均值 |
| NDCG | 考虑排序位置的相关性得分 |

### 生成质量

| 维度 | 说明 |
|------|------|
| 上下文相关性 | 检索内容精准度 |
| 答案忠实度 | 答案是否忠于上下文 |
| 答案相关性 | 是否直接回答问题 |

### 鲁棒性

| 能力 | 说明 |
|------|------|
| 噪声鲁棒性 | 管理无关文档的能力 |
| 负面拒绝 | 无相关知识时能拒绝回答 |
| 信息整合 | 多文档综合信息能力 |
| 反事实鲁棒性 | 识别已知不准确信息 |

---

## 十一、要点总结

1. **RAG = 检索 + 生成**：先找相关资料，再基于资料回答问题
2. **从朴素到模块化**：Naive → Advanced → Modular，灵活性逐级提升
3. **REALM 是 RAG 的预训练版本**：把检索器也参与训练，端到端优化
4. **检索是核心瓶颈**：质量直接影响生成效果
5. **RAG 和微调互补**：RAG 动态更新知识，微调深度定制行为
