# Day 5: LLM基础

## 第一部分：AI Agent

### 一、什么是 AI Agent

AI Agent 是一个**自主决策系统**：设定目标后，根据观察产生行动，影响环境，再接收新观察，形成闭环循环。
![4fcd2420843e254ee8fc82b9c50634d0.jpg](https://i.postimg.cc/NG879nzL/4fcd2420843e254ee8fc82b9c50634d0.jpg)
### 二、Agent 与 LLM 的结合

| 方式 | 机制 | 特点 |
|------|------|------|
| 强化学习（RL） | 通过奖励信号反馈 | 反馈信息有限，仅有数值 |
| LLM Agent | 通过语言输入→语言输出执行 | 反馈包含丰富上下文信息 |

### 三、记忆管理（Memory）

Agent 记忆过多观察时，容易关注不相关信息导致错误决策。解决方案类似 RAG：

- **存储**：将记忆存到 Memory 中
- **检索**：当前观察作为 Query，检索最相关记忆用于决策
- **质量筛选**：用模型记录高质量数据
- **反思整理**：用 Reflection（模型）整理有效记忆，建立记忆间关系
![56e3420d884fa95aa0e0ce982ef0e657.jpg](https://i.postimg.cc/XqDvB4SN/56e3420d884fa95aa0e0ce982ef0e657.jpg)
### 四、关键发现

- **正面例子 > 负面例子**：给 Agent 看"要做什么"比"不要做什么"更有效，也比全部给效果好
- **Function Call**：LLM 使用工具（搜索引擎、Python、其他 AI 等），只需知道工具的输入输出，无需了解内部细节

### 五、内外知识冲突

LLM 做 RAG 时会比较内外知识：
- **内外知识差距越大，越不相信外部知识**
- LLM 对信息的倾向：AI 生成的内容 > 人工内容；时间较近 > 时间较早

### 六、规划能力强化

| 阶段 | 做法 | 问题 |
|------|------|------|
| 实际尝试 | 多种方法并行尝试 | 步骤太多 |
| 剪枝 | 放弃不切实际的路径 | 部分行为无法撤回 |
| 模拟评估 | 在模拟环境中测试并反馈 | 可能想太多，放弃操作 |

### 七、功能向量（Steering Vector）

向量在某一层可能呈现特定功能，加减该向量会产生完全不同结果：

- **发现方法**：含功能向量的多组结果平均值 − 不含功能向量的结果平均值
- **寻找工具**：Sparse Auto-Encoder（SAE）
- **典型例子**：否定向量、谄媚向量等

## 第二部分：语言模型发展史

### 一、四阶段演进

| 阶段 | 代表 | 核心特征 |
|------|------|----------|
| **统计语言模型（SLM）** | N-gram | 马尔可夫假设，辅助特定任务 |
| **神经语言模型（NLM）** | word2vec | 神经网络表征，词向量+上下文 |
| **预训练语言模型（PLM）** | ELMo, BERT | 大规模预训练→微调适配下游 |
| **大型语言模型（LLM）** | GPT, LLaMA | 千亿参数，通用任务处理 |

**趋势**：早期关注建模和文本生成 → 现在关注复杂问题解决。

### 二、关键模型

| 模型 | 技术 | 意义 |
|------|------|------|
| word2vec (CBOW/Skip-gram) | 浅层神经网络 | 分布式词表示 |
| ELMo | 预训练 BiLSTM | 上下文感知 |
| BERT | Transformer + 无标签预训练 | 通用语义特征+微调范式 |

### 三、Few-shot 提示方式

| 方式 | 做法 |
|------|------|
| Zero-shot | 不给实例，直接输入指令 |
| One-shot | 给一个例子辅助推断 |
| Few-shot | 给少量例子辅助推断 |

## 第三部分：规模定律（Scaling Laws）

### 一、KM 规模定律

模型规模 $N$、数据集规模 $D$、训练计算量 $C$ 与损失的关系：

$$
L(N) = \left( \frac{N_c}{N} \right)^{\alpha_N}, \quad
L(D) = \left( \frac{D_c}{D} \right)^{\alpha_D}, \quad
L(C) = \left( \frac{C_c}{C} \right)^{\alpha_C}
$$

### 二、损失的分解

| 类型 | 来源 | 说明 |
|------|------|------|
| **不可约损失 $E$** | 数据分布的熵 | 语言本身的不确定性（歧义、多答案） |
| **可约损失** | 模型与真实分布的 KL 散度 | 模型性能问题，可通过扩大规模减少 |

### 三、Chinchilla 规模定律

$$L(N, D) = E + \frac{A}{N^{\alpha}} + \frac{B}{D^{\beta}}$$

| 符号 | 含义 | 拟合值 |
|------|------|--------|
| $E$ | 不可约损失（天花板） | 1.69 |
| $\frac{A}{N^{\alpha}}$ | 模型不足带来的可约损失 | $A=406.4, \alpha=0.34$ |
| $\frac{B}{D^{\beta}}$ | 数据不足带来的可约损失 | $B=410.7, \beta=0.28$ |

**最优配置**：
$$
N_{opt}(C) = G \left( \frac{C}{6} \right)^a, \quad
D_{opt}(C) = G^{-1} \left( \frac{C}{6} \right)^b
$$
其中 $a = \frac{\alpha}{\alpha + \beta}, b = \frac{\beta}{\alpha + \beta}$

### 四、KM vs Chinchilla

| | KM | Chinchilla |
|------|------|------|
| 倾向 | 更多预算分配给模型规模 | 模型和数据**同比例**增加 |
| 依据 | — | $\alpha \approx 0.34$, $\beta \approx 0.28$，接近 |
| 启示 | 大模型优先 | 数据和模型同样重要 |

### 五、规模定律的用途

- **预测**：基于小模型预测大模型表现，避免全量尝试的时间损耗
- **监测**：用于监测大模型的训练状态

## 第四部分：涌现能力（Emergent Abilities）

### 一、定义

涌现能力是**区分 LLM 与 PLM 的最显著特征**：在小模型中不存在，模型规模达到某一水平时性能显著突破。

### 二、三种典型涌现能力

| 能力 | 说明 |
|------|------|
| **上下文学习（ICL）** | 通过完成输入文本序列生成输出，无需额外训练 |
| **遵循指令** | 微调后在未见过的任务上也能良好表现，提升泛化 |
| **逐步推理（CoT）** | 利用链式思维提示，通过中间推理步骤解决复杂问题 |

### 三、重要区分

| 维度 | 衡量方式 | 可预测性 |
|------|----------|----------|
| 规模定律 | 语言建模损失 | 可预测（平滑） |
| 涌现能力 | 任务性能 | 不可预测（断崖式） |

### 四、关于涌现的争论

断崖式增长可能是因为**粗粒度评估指标**导致的表象，增加评估分辨率后可能表现为平滑增长。

---

## 五、要点总结

1. **AI Agent = 观察 → 决策 → 行动 → 再观察**，LLM 提供丰富上下文反馈
2. **Memory + RAG 式检索** 是 Agent 长时记忆的关键
3. **功能向量** 揭示了模型的内部表示结构，可通过 SAE 发现
4. **语言模型四阶段**：SLM → NLM → PLM → LLM，从辅助工具到通用任务处理器
5. **Chinchilla 定律**：模型和数据应同比例扩大，而不是一味堆参数
6. **涌现能力**：ICL、指令遵循、CoT 推理是 LLM 的核心突破
