# Day 6: LLM 训练原理与机制深度解析

## 第一部分：LLM 训练三阶段

### 一、训练流程

| 阶段 | 目标 | 本质 |
|------|------|------|
| **Pre-train** | 获得预测下一个 token 的能力 | 学会语言模式 |
| **SFT**（监督微调） | 告诉模型"某种输出是正确的" | 学会回答格式 |
| **RLHF**（人类反馈强化学习） | 告诉模型"什么样的回答是好的" | 学会回答质量 |

SFT + RLHF 合称 **Alignment（对齐）** 或 Fine-tune。模型名中 `base` 表示只做了预训练，`chat`/`instruct` 表示做过对齐。

### 二、知识蒸馏（Knowledge Distillation）

将问题送给训练好的模型回答 → 筛选高品质输出 → 用于对另一模型的微调 → 大幅提升该模型的能力。

### 三、有效 Pre-train 的关键

#### 无效训练实验

- Pre-train：准备 N 个人的资料，每人只出现一次
- Alignment：抽出 N/2 个人制造相关问题
- 测试：剩下 N/2 个人
- 结果：失败。因为 Pre-train 本质是填词训练，资料不足形成**固定推理模式**，无法进行多样化判断

#### 有效训练

- 每个或一部分人准备多种版本资料
- 核心原则：模型的**多样性比重复的资料更重要**

### 四、Alignment 的关键边界

- 在 Alignment 阶段学习**模型不知道的知识**会破坏模型，导致整体正确率下降
- Align 的核心作用是**告诉模型怎么回答**，而非教它新知识
- Pre-train 看到不该看的知识后**难以真正清除**
- RL 是 Alignment 的好方法
- 对齐数据**质量 > 数量**，alignment 前后模型行为差别不大

## 第二部分：后训练与灾难性遗忘

### 一、Post-Training（Continual Learning）

基础模型（base/chat/instructed）→ Post-Training → Fine-tuned Model

用于学习新知识。核心挑战：后训练后可能丢失原本能力（尤其是 safety），即**灾难性遗忘**。

### 二、灾难性遗忘的规律

- **模型学得越多/越好，越容易遗忘**
- 使用 LoRA 可在一定程度上避免遗忘

### 三、缓解策略

| 策略 | 做法 | 前提条件 |
|------|------|----------|
| 原始数据重训 | 用丢失能力相关的原数据训练 | 需原始训练数据 |
| 混合训练 | 后训练数据中掺入原相关能力数据 | 同上 |
| **Pseudo（Experience Replay）** | 让模型自己生成其看过的数据用于训练 | 无需原始数据 |
| **Paperphrase** | 将问题转化为与学习资料匹配的格式 | 模型改造输入 |
| **Self-output** | 正确自答训练自己，错误用人写答案 | 类似 RL-based Training |

## 第三部分：数据准备理论

![数据理论框架](https://i.postimg.cc/VkL44XHW/ping-mu-jie-tu-2026-07-06-093531.png)

### 一、数据混合（Data Mixture）

**核心发现**：性能不仅与数据量有关，也与数据的**异质性**有关——多来源数据（文本、书籍等）效果更优。

#### 1. 多任务学习（MTL）视角

泛化界：
$$\mathcal{R}(f_1,...,f_n) \leq \hat{\mathcal{R}}(f_1,...,f_n) + \sqrt{\frac{(l(E) + l_E(f_1,...,f_n)) \log(2) + \log(1/\delta)}{2mn}}$$

- 泛化上界**不依赖模型庞大的原始参数量**，而是依赖联合编码所有 n 任务模型所需的**压缩编码长度** $l(E) + l_E(f_1, ..., f_n)$
- $m$：每个任务的样本量；$n$：任务数
- **样本数量少时，任务数足够多也能有效抑制泛化上界**
- 参数量过大理论上会过拟合，但因共享参数可实现更强泛化

#### 2. 多源学习（MSL）视角

平均总变差误差 $R_{TV}$ 由条件分布空间的复杂性控制。当源分布具有足够**参数相似性**时，多源训练保证比单独在各源上训练获得**更精确的误差界**。

#### 3. 数据混合预测模型

$$L_i(r_i, s) = \frac{A_i}{r_i^{\alpha_i}} \left( \frac{B_i}{s^{\beta_i}} + C_i \right)$$

| 符号 | 含义 |
|------|------|
| $L_i(r_i, s)$ | 第 i 个数据域的**验证损失** |
| $r_i$ | 第 i 域在训练数据中的**混合比例**（0 < r ≤ 1） |
| $s$ | 总**训练步数** |
| $A_i$ | 该域的**尺度参数**（反映基础难度） |
| $\alpha_i$ | **比例缩放指数**（比例越低损失越高） |
| $B_i$ | **步数缩放参数** |
| $\beta_i$ | **步数衰减指数**（随训练步数幂律下降） |
| $C_i$ | **不可约误差常数**（即使无限数据和训练也无法消除） |

**关键洞察**：
- 某域占比越小，损失越大（$\frac{A_i}{r_i^{\alpha_i}}$）
- 随训练增加，损失下降但收敛到 $C_i$
- 即使某领域数据量少，也可通过**延长步数**提升效果

#### 4. 优化策略

| 方法 | 核心思路 |
|------|----------|
| **REGMIX** | 定义为回归任务，数据混合的相对优越性在不同模型规模和数据量下保持**不变** |
| **UtiliMax** | 在三个因素间实现最佳平衡：每个数据源的效用、多样性、规模 |
| **DoReMi** | 利用最小-最大优化，最小化最差情况损失 |

### 二、数据去重与筛选

- 数据去重解决了**不必要的记忆问题**，减少了模型记忆训练文本的频率
- **数据重复是导致记忆产生并带来隐私风险的关键驱动因素**

| 阶段 | 方法 | 特点 |
|------|------|------|
| 早期 | 最大噪声子集去重 | 受计算约束 |
| 优化 | 全局 MinHash LSH | 过于缓慢，基于 CPU |
| 突破 | **FED 框架** | 低成本可重用哈希函数 + GPU 并行优化 |
| 最新 | **SoftDedup** | 软重加权机制，解决硬去重的信息丢失风险 |

### 三、记忆机制

#### 马赛克记忆

LLM 的记忆**不是逐字回忆**，而是通过整合训练语料中部分重叠或相似的序列（即模糊重复）来**拼凑记忆**。

#### 对抗压缩比（ACR）

核心思想：如果训练序列可以通过**明显短于原序列的提示**被完整恢复，就说明被记忆了。

## 第四部分：推理能力

### 一、推理的本质转变

推理能力不再被视为模型参数的静态属性，而是作为在交互过程中**分配的计算资源的动态函数**。主要通过 **CoT** 机制以及各种基于外部搜索的算法来建立，扩展了模型的"思考"过程。

### 二、CoT 的深度理解

- **CoT 可作为自回归模型的有效深度扩展器**
- 底层负责"读懂"问题与上下文，上层负责"写出"推理与答案
- 推理不是简单地"记住更多数据"，而是需要**训练过程穿透到更深的优化景观中**（过拟合后继续训练）才能涌现出来

### 三、理论保证

- CoT 推理不是"碰巧"出现的，梯度下降优化在**足够样本下能够保证学到**——从工程观察到理论保证
- 推理能力不是"被注入"的，而是**预训练已经赋予模型潜能**——只需通过特定解码方式"激活"
- 但同时 CoT 本质**脆弱**：严重依赖训练推理路径与测试查询的一致性，在复杂环境中可能是高级模式匹配

### 四、过度思考（Over-thinking）

模型在本质上简单或无法解决的任务上生成过多、多余甚至错误的推理步骤。

### 五、潜在推理（Latent Reasoning）

将理论焦点从显式 token 级 CoT 转移到内部状态级计算：

- **COCONUT**：在连续潜在空间中进行推理，将隐藏状态反馈作为后续输入，能同时编码多个推理分支
- **优势**：不受显式 CoT 序列长度瓶颈的限制

## 第五部分：评估与安全性

![评估框架](https://i.postimg.cc/gkRwTKJw/ping-mu-jie-tu-2026-07-06-142450.png)

### 一、基准废弃问题

- 传统基准快速**失效**——顶级模型分数接近饱和，无法有效区分 SOTA
- **一个总分掩盖了任务所需技能的组合复杂性**：两个模型分数相同，但能力结构完全不同
- 基准测试是经验性的：能展示已知分布上的失败，但**不能保证未知输入下的行为**

### 二、LLM 作为评审

- 仅具有**表面效度**——表现了效度但不代表真正有效
- 两关键缺陷：**低可靠性**和**较差的心理测量有效性**
- 存在多种系统性偏差（更喜欢特定位置的答案等）

### 三、幻觉

#### 产生机制

| 机制 | 说明 |
|------|------|
| 知识压制 | 生成过程中占主导的知识会抑制不那么频繁的知识 |
| 架构限制 | Transformer 在执行函数组合方面存在固有限制 |
| 子序列嵌入 | 仅解码器 Transformer 中，主导输入子序列触发错误输出 |
| 评估反噬 | 训练后基准测试惩罚不确定性，实际上鼓励模型**猜测而非回避** |

#### 缓解方案

| 方案 | 原理 |
|------|------|
| 负面反馈 | 实现具有广度的一致生成 |
| 对比解码 | 放大被掩盖的知识以缓解偏差 |
| 修改评估 | 奖励适当表达的不确定性 |

### 四、可解释性

| 类型 | 目标 |
|------|------|
| **全局** | 模型学到了什么，语言/语义信息如何组织 |
| **局部** | 单个预测的解释 |
| **机制** | 内部计算机制的原理 |

### 五、线性表示假设

高层语义概念在 LLM 的激活空间中**以线性方向的形式编码**。

### 六、典型失败模式

- **反向知识失败**：学会"A 是 B"不会自动泛化到"B 是 A"
- **位置偏差**：对长输入的开头和结尾赋予更高权重

---

## 六、要点总结

1. **训练三阶段**：Pre-train（学会模式）→ SFT（学会格式）→ RLHF（学会质量），对齐不教新知识
2. **数据多样性 > 数量**：同主题多版本比广撒网更有效，对齐质量 > 对齐数量
3. **灾难性遗忘**：学得越好越易忘，LoRA/Pseudo/Self-output 可缓解
4. **数据混合可预测**：损失与比例、步数呈幂律关系，可通过延长步数弥补数据不足
5. **推理是涌现的**：预训练赋予潜能，CoT 激活但本质脆弱（可能是高级模式匹配）
6. **评估正在失效**：静态基准饱和，幻觉源于知识压制机制，对比解码可部分缓解
